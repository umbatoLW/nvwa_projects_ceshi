# 剧本格式兼容方案

## 问题分析

用户上传的剧本格式千奇百怪，当前的正则匹配模式可能无法覆盖所有情况。

## 当前支持的格式

### 集数标记
- `【第1集】标题` - 书名号格式（AI生成）
- `[第1集]标题` - 方括号格式
- `第1集` - 简单格式
- `第1集：标题` - 带冒号
- `EPISODE 1`、`Episode 1`、`E1` - 英文格式
- `1. 标题` - 数字序号

### 剧本正文标记
- `【剧本正文】`、`【正文】`
- `[剧本正文]`、`[正文]`

---

## 需要新增支持的格式

### 集数标记（扩展）

| 格式 | 正则 | 说明 |
|------|------|------|
| `第一集` | `第[一二三四五六七八九十百]+集` | 中文数字 |
| `第1话` | `第\s*(\d+)\s*话` | 日剧格式 |
| `第1章` | `第\s*(\d+)\s*章` | 小说改编 |
| `EP01` | `EP\s*0*(\d+)` | 英文补零 |
| `#1` | `#\s*(\d+)` | 井号标记 |
| `P1` | `P\s*(\d+)` | Part缩写 |

### 剧本正文标记（扩展）

| 格式 | 正则 | 说明 |
|------|------|------|
| `剧本正文：` | `剧本正文[：:]` | 带冒号 |
| `---正文---` | `[-]{3,}正文[-]{3,}` | 分隔线 |
| `SCRIPT` | `^SCRIPT$` | 英文标记 |

### 场景格式（扩展）

| 格式 | 正则 | 说明 |
|------|------|------|
| `1-1`、`1-2` | `(\d+)-(\d+)` | 集数-场景数 |
| `S1E1` | `S\s*(\d+)\s*E\s*(\d+)` | 季-集 |

---

## 解决方案

### 方案一：增强正则模式库（推荐）

**优点**：无需AI调用，响应快，成本低
**缺点**：仍可能有遗漏格式

```javascript
const episodePatterns = [
  // 书名号格式（最高优先级）
  /^【第\s*(\d+)\s*集】[：:：\s]*(.*)$/i,
  /^【第\s*(\d+)\s*话】[：:：\s]*(.*)$/i,
  /^【第\s*(\d+)\s*章】[：:：\s]*(.*)$/i,
  
  // 方括号格式
  /^\[第\s*(\d+)\s*集\][：:：\s]*(.*)$/i,
  /^\[EP\s*0*(\d+)\][：:：\s]*(.*)$/i,
  
  // 中文数字
  /^第([一二三四五六七八九十百]+)集[：:：\s]*(.*)$/i,
  
  // 简单格式
  /^第\s*(\d+)\s*集$/i,
  /^第\s*(\d+)\s*话$/i,
  /^第\s*(\d+)\s*章$/i,
  
  // 带标题格式
  /^第\s*(\d+)\s*集[：:：\s]+(.*)$/i,
  /^第\s*(\d+)\s*话[：:：\s]+(.*)$/i,
  /^第\s*(\d+)\s*章[：:：\s]+(.*)$/i,
  
  // 英文格式
  /^EPISODE\s*(\d+)[：:：\s]*(.*)$/i,
  /^Episode\s*(\d+)[：:：\s]*(.*)$/i,
  /^EP\s*0*(\d+)[：:：\s]*(.*)$/i,
  /^E\s*(\d+)[：:：\s]*(.*)$/i,
  
  // 季集格式
  /^S\s*(\d+)\s*E\s*(\d+)[：:：\s]*(.*)$/i,
  
  // 数字序号
  /^(\d+)[\.．]\s*(.*)$/,
  /^#(\d+)[：:：\s]*(.*)$/i,
  /^P\s*(\d+)[：:：\s]*(.*)$/i,
];
```

### 方案三：用户手动指定格式

在分集失败时，提供格式选择：


### 方案四：智能检测 + 多模式尝试

```javascript
function detectScriptFormat(content: string): string {
  // 检测关键字频率
  const keywords = {
    '第X集': (content.match(/第\d+集/g) || []).length,
    '【第X集】': (content.match(/【第\d+集】/g) || []).length,
    'EPX': (content.match(/EP\s*\d+/gi) || []).length,
    '第X话': (content.match(/第\d+话/g) || []).length,
    '数字序号': (content.match(/^\d+\./gm) || []).length,
  };
  
  // 返回频率最高的格式
  return Object.entries(keywords)
    .sort((a, b) => b[1] - a[1])[0][0];
}
```

## 推荐实施方案

**三重保障机制**：

1. **第一层：增强正则模式库**
   - 覆盖90%+常见格式
   - 按优先级匹配

2. **第二层：智能检测**
   - 分析内容特征
   - 动态调整匹配策略

3. **第三层：AI兜底**
   - 当正则失败时调用AI
   - 提取结构信息

4. **第四层：用户干预**
   - 提供格式选择选项
   - 手动指定分集方式

---

## 实施优先级

| 优先级 | 方案 | 成本 | 效果 |
|--------|------|------|------|
| P0 | 增强正则模式库 | 低 | 覆盖90%+ |
| P1 | 智能检测 | 低 | 提升5% |
| P2 | 用户手动指定 | 中 | 兜底方案 |
| P3 | AI辅助识别 | 高 | 最后兜底 |
